## 只保留 3 张表：Actor / Card / Edge

### 1) `actor`（谁在出牌 / 谁在裁决）

* `actor_id`
* `type`（you / human / ai / env）
* `name`

### 2) `card`（唯一账本：所有东西都写成“牌”）

> 关键：**不分 evidence/inference/action/verdict 多表**，统一都叫 card，用 `kind` 区分。你用 view/查询去“分牌堆”，别用物理多表去分世界观。

最小字段：

* `card_id`
* `kind`（signal / inference / action / oracle / kill / contract …）
* `round_id`（可空；有就表示属于某一轮对局壳）
* `actor_id`
* `payload_json`（你要的组合、结构、评分、解释、任何东西都塞这里）
* `hash`（钉死内容，防自己后改）
* `status`（active / killed / archived）
* `created_at`
* `supersedes_id`（可空；做版本跃迁：新牌替旧牌）

> 你怕字段痛苦，就把复杂度塞进 `payload_json`，把“结构硬约束”压到 **kind + hash + status + supersedes** 这几个硬骨头上。

### 3) `edge`（引用式结构：牌和牌之间怎么连）

* `edge_id`
* `src_card_id`
* `dst_card_id`
* `kind`（uses / supports / contradicts / targets / derived_from）
* `created_at`

## 你那条闭环在这 3 张表里的落点

* **结果片段 fakecards**
  = `card.kind = signal`（payload 里标记 facet=fake / partial 都行）
  片面就片面，片面是事实，不用修饰。

* **后验态势账本 player_cards**
  = `card.kind = inference`
  用 `edge(derived_from/uses)` 指向你收集的 signal，以及你当时依赖的 contract（如果有）。

* **两套依据组合出牌（博弈） → 你的/我的 action**
  = `card.kind = action`
  用 edge 指向你当时采用的 inference + contract（你说的 chains）。

* **现实 oracle（裁决）**
  = `card.kind = oracle`
  payload 里放 `pass_flag/score/diff/exposure` 等；用 edge 指向 action（targets）和裁决依据（uses：现实信号）。

* **输的被猎杀**
  你要黑暗森林味道，就别“更新 status”当主语义，而是**写一张 kill 牌**：
  = `card.kind = kill`，edge.targets 指向被猎杀对象（inference / action / contract / 甚至某个 oracle 口径）
  同时把被猎杀牌 `status = killed`（这是派生状态，不是叙事核心）。

这一下你想要的特征全保留了，而且没有“多表地狱”。

## 为什么这版更像你的轴，而不是我之前那版

* 黑暗森林里最怕“解释表”“证据表”这种带价值判断的命名，它会把你拖回文明世界的语境。
  你这里先统一叫 **card**，只区分 **kind**，价值判断交给 oracle，不写死在表名里。

* 你要猎杀的是“用法/世界线/口径/黑盒”，不是只猎杀“结果”。
  统一 card 之后，**任何东西都能被 kill**，系统天然闭环。
